# Connector Packaging and Import Protocol

## File Layout per Connector Operation

```
src/main/resources/                    # NO Maven filtering
├── {connector-id}.def                 # EMF ConnectorDefinition XML
├── {connector-id}.properties          # i18n labels for Studio UI
└── connector.png                      # 16x16 valid binary PNG icon

src/main/resources-filtered/           # Maven filtering ENABLED
└── {connector-id}.impl                # Implementation descriptor (${variables} resolved)
```

## .impl File Pattern

```xml
<?xml version="1.0" encoding="UTF-8"?>
<implementation:connectorImplementation
    xmlns:implementation="http://www.bonitasoft.org/ns/connector/implementation/6.0">
  <implementationId>{connector-id}-impl</implementationId>
  <implementationVersion>${project.version}</implementationVersion>
  <definitionId>{connector-id}</definitionId>
  <definitionVersion>1.0.0</definitionVersion>
  <implementationClassname>com.bonitasoft.connectors.{package}.{Class}Connector</implementationClassname>
  <hasSources>false</hasSources>
  <description>{connector-id} connector implementation.</description>
${connector-dependencies}
</implementation:connectorImplementation>
```

### Critical .impl Rules
- `definitionId` MUST match .def `<id>` exactly
- `definitionVersion` MUST match .def `<version>` exactly
- `implementationClassname` must exist in the compiled JAR
- `${connector-dependencies}` is auto-generated by `dependencies-as-var.groovy`
- `${project.version}` is resolved by Maven resource filtering

## Groovy Dependency Script

Located at `src/script/dependencies-as-var.groovy` (project root):

```groovy
import groovy.xml.MarkupBuilder

def xml = new StringWriter()
def builder = new MarkupBuilder(xml)
builder.jarDependencies {
    jarDependency("${project.artifactId}-${project.version}.${project.packaging}")
    project.artifacts
            .findAll { artifact -> artifact.scope == "compile" || artifact.scope == "runtime" }
            .sort { artifact -> artifact.artifactId }
            .each { artifact ->
                jarDependency("${artifact.artifactId}-${artifact.version}.${artifact.type}")
            }
}
def deps = xml.toString()
project.properties.setProperty("connector-dependencies", deps)
```

Invoked by `groovy-maven-plugin` in `generate-resources` phase.

## POM Configuration (per operation module)

```xml
<build>
    <resources>
        <resource>
            <directory>src/main/resources</directory>
            <filtering>false</filtering>
        </resource>
        <resource>
            <directory>src/main/resources-filtered</directory>
            <filtering>true</filtering>
        </resource>
    </resources>
    <plugins>
        <!-- 1. Generate ${connector-dependencies} from Maven dependency tree -->
        <plugin>
            <groupId>org.codehaus.gmaven</groupId>
            <artifactId>groovy-maven-plugin</artifactId>
        </plugin>
        <!-- 2. Produce -bonita.jar fat JAR with all deps shaded -->
        <plugin>
            <groupId>org.apache.maven.plugins</groupId>
            <artifactId>maven-shade-plugin</artifactId>
        </plugin>
    </plugins>
</build>
```

## Shade Plugin Configuration (parent pluginManagement)

```xml
<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-shade-plugin</artifactId>
    <version>3.5.1</version>
    <configuration>
        <createDependencyReducedPom>false</createDependencyReducedPom>
        <shadedArtifactAttached>true</shadedArtifactAttached>
        <shadedClassifierName>bonita</shadedClassifierName>
        <transformers>
            <transformer implementation="org.apache.maven.plugins.shade.resource.ServicesResourceTransformer"/>
        </transformers>
        <filters>
            <filter>
                <artifact>*:*</artifact>
                <excludes>
                    <exclude>META-INF/*.SF</exclude>
                    <exclude>META-INF/*.DSA</exclude>
                    <exclude>META-INF/*.RSA</exclude>
                </excludes>
            </filter>
        </filters>
        <artifactSet>
            <excludes>
                <exclude>org.bonitasoft.engine:*</exclude>
                <exclude>org.slf4j:*</exclude>
            </excludes>
        </artifactSet>
    </configuration>
    <executions>
        <execution>
            <phase>package</phase>
            <goals><goal>shade</goal></goals>
        </execution>
    </executions>
</plugin>
```

## All-in-One Module (multi-connector)

For multi-module projects, the `-all` module:
- Depends on ALL operation modules
- Shade plugin merges all .def, .impl, .properties, .png from dependency JARs
- No need for own connector resources (inherits from modules)
- MUST be LAST in parent POM `<modules>` list
- Must use `shadedArtifactAttached=true` + `shadedClassifierName=bonita`

## Import into Bonita Studio 2021.2+

1. Build: `mvn clean install -DskipTests`
2. Studio: **Extensions > Import extension**
3. Select the `-bonita.jar` file
4. If not visible after import: close/reopen project or restart Studio

## Post-Build Verification

```bash
# 1. Check resources at JAR root
jar tf target/*-bonita.jar | grep -E '\.(def|impl|properties|png)$' | grep -v META-INF

# 2. Check .impl resolved (not placeholder)
jar xf target/*-bonita.jar {id}.impl && grep jarDependenc {id}.impl && rm {id}.impl

# 3. Check connector classes
jar tf target/*-bonita.jar | grep "Connector.class"

# 4. Check JAR size (multi-MB expected)
ls -lh target/*-bonita.jar
```

## Common Errors

| Symptom | Cause | Fix |
|---------|-------|-----|
| `eResource() null` | Invalid EMF in .def | See `emf-definition-rules.md` |
| Imported as library | Missing .def/.impl at JAR root | Verify shade includes resources from deps |
| `${connector-dependencies}` literal in JAR | .impl in wrong directory | Move to `src/main/resources-filtered/` |
| Empty -bonita.jar | Reactor deps not resolved | Run `mvn install` from root, not `-pl` alone |
| No category | `<category>` not self-closing | Fix to `<category icon="x" id="y"/>` |
